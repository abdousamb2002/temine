# Temine 竞品对比分析

> AI 编程终端编排器 — 一个全新的产品品类

## 核心定位

tmux 管理窗格，iTerm2 提升终端，Warp 让 AI 帮你写命令，Cursor 让 AI 帮你写代码 — 而 **Temine 管理你的所有 AI 代理**。

## 市场空白

当开发者同时运行 4-8 个 Claude Code 实例时，没有任何现有工具能够：
- 一键将它们排布到屏幕上
- 实时感知哪个实例在等你确认
- 在浏览器中总览所有实例状态
- 在 AI 等你时推送系统通知

## 各产品详细分析

### tmux / GNU Screen — 传统终端复用器

- **定位**: 通用终端会话复用
- **优势**: 极其稳定、全平台、SSH 远程会话保持、可高度定制
- **劣势**: 学习曲线陡峭、无图形界面、无 AI 感知、所有窗格挤在同一终端内
- **与 Temine 的区别**: tmux 是"一个终端内管多个虚拟窗格"，Temine 是"管理操作系统级的多个独立终端窗口"

### iTerm2 — macOS 高级终端

- **定位**: macOS 最强终端模拟器
- **优势**: 功能极其丰富、Python 脚本 API、Triggers 可做基础模式匹配、免费开源
- **劣势**: 配置复杂、Triggers 需手动配规则、无法从外部批量管理多窗口排布和状态
- **与 Temine 的区别**: iTerm2 是"更好的终端"，Temine 是"从外部编排多个终端"

### Warp — AI 终端

- **定位**: AI 原生的现代终端
- **优势**: 现代 UI 极佳、AI 命令生成实用、Block 设计让输出更可读、团队协作
- **劣势**: 需要登录（隐私顾虑）、部分功能付费、无法管理多个 Claude Code 实例
- **与 Temine 的区别**: Warp 是"AI 帮你写命令"，Temine 是"管理多个 AI 代理的运行状态"

### Zellij — 现代终端复用器

- **定位**: tmux 的现代化替代品（Rust 编写）
- **优势**: 比 tmux 易学、Rust 高性能、Layout 预定义、WASM 插件可扩展
- **劣势**: 生态不成熟、仍是虚拟窗格、无 AI 支持、无 Web 面板
- **与 Temine 的区别**: Zellij 是"更好的 tmux"，Temine 是完全不同的品类

### Claude Code CLI 本身

- **定位**: Anthropic 官方 AI 编程 CLI
- **优势**: 官方工具、功能强大、持续迭代
- **劣势**: **完全没有多实例管理能力**、无窗口编排、无跨实例状态可视化、等待确认时没有通知
- **与 Temine 的关系**: Temine 是 Claude Code 多窗口并行工作时的**必备伴侣工具**

### Cursor / Windsurf — AI IDE

- **定位**: AI-first 代码编辑器
- **优势**: AI 编辑体验好、代码上下文感知强、一站式体验
- **劣势**: 终端是附属品、多终端视图空间受限、无法管理 IDE 外部的 CLI 工具
- **与 Temine 的区别**: AI IDE 把终端当配角，Temine 把终端当主角

## Temine 五大差异化优势

### 1. 唯一的 "AI CLI 编排层"

不是终端模拟器，不是终端复用器，不是 IDE。是一个**终端编排器** — 在操作系统层面管理多个终端窗口。这个定位在市场上独一无二。

### 2. AI 语义级状态检测

多层规则 + 置信度评分，能识别 Claude Code 的权限提问、确认提示、Spinner 动画、进度条、错误和完成信号。远超 iTerm2 Triggers 的简单正则。

### 3. Web 控制面板 = "AI 任务控制中心"

在浏览器中实时看到所有终端状态和输出预览，一键调整布局，互换窗口，管理记录。SSE 实时推送。这是所有竞品都不具备的"上帝视角"。

### 4. 零依赖极简架构

`"dependencies": {}` — 完全零依赖。HTTP 用 `node:http`，文件用 `node:fs`，终端控制用 `osascript`。安装极快、无供应链风险、无版本冲突。

### 5. 不侵入终端本身

通过 AppleScript 从外部读取和控制终端窗口，不修改 shell 配置、不注入代码。用户继续使用习惯的 Terminal.app 或 iTerm2。

## 局限性

| 局限 | 原因 | 潜在改进 |
|------|------|---------|
| 仅 macOS | AppleScript 是 macOS 独有 | 未来可考虑 Linux xdotool |
| 被动轮询检测 | 通过轮询终端内容而非 AI 主动推送 | 期待 Claude Code 提供状态 hook |
| 无 SSH 持久化 | 不是 tmux 替代品 | 定位不同，管理本地窗口 |
| 无自动回复 | 仅监控通知，不能自动确认 | 可考虑添加远程确认 |
